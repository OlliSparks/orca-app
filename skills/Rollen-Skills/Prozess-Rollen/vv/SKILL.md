# VV - Versorgungsverantwortliche

Rolle für Versorgungsverantwortliche (After Sales, Classic).

## Steckbrief

| Eigenschaft | Wert |
|-------------|------|
| **Kürzel** | VV |
| **Englisch** | Supply Responsible |
| **Typ** | Process |
| **Launch Phase** | LP1 |
| **Abbildung** | Über "ABO" (Abonnement) |

## Berechtigungen

| Berechtigung | Beschreibung |
|--------------|--------------|
| **View** | Lesezugriff |
| **Approve** | Genehmigungsrechte |

## Abbildung über ABO

Diese Rolle wird nicht als feste Gruppenmitgliedschaft abgebildet, sondern über das **Abonnement-System (ABO)**. Benutzer mit besonderem Interesse an bestimmten Assets/Prozessen können sich diese abonnieren.

## Kontext

- **After Sales**: Nachbetreuung und Ersatzteilversorgung
- **Classic**: Klassik-Fahrzeuge und deren Versorgung

## Hauptaufgaben

1. **Versorgungssicherung** - Sicherstellung der Ersatzteilversorgung
2. **Prozessüberwachung** - Monitoring relevanter Prozesse
3. **Genehmigung** - Freigabe von Anfragen im Versorgungsbereich

## Workflow-Beteiligung

Der VV wird bei relevanten Prozessen automatisch über das ABO-System benachrichtigt und kann bei Bedarf Genehmigungen erteilen.
